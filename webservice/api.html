URI base

http://localhost/aw12/webservice

API

A API é apresentada em função de cada recurso.

Em todos os recursos o método GET está definido, podendo receber como paramêtro "text/xml" ou "json" no campo Accept do pedido HTTP, para que a resposta seja em XML ou JSON, respectivamente. Também são aceites como query variables as variáveis start e count. start indica onde os elementos retornados começam, e count indica quantos elementos devem ser devolvidos na resposta. Se não houverem elementos suficientes disponíveis para perfazer count, todos os elementos disponíveis são retornados. Obviamente tanto count como start devem ser sempre valores inteiros e positivos. Se estas variáveis não estiverem definidas, o valor por defeito utilizado é 0 para start e 10 para count.

/noticias.php

Este é o recurso relativo as notícias.

Para este recurso estão definidos os métodos GET e POST.

GET
Devolve uma lista de notícias.
POST
Permite inserir uma nova notícia, que deve ser passada no corpo do pedido HTTP, em XML, respeitando o XML Schema noticia.xsd. Como pode ser visto no XML Schema, a notícia a ser inserida já deve conter todas as relações existentes.

/noticias.php/{id}

Este recurso é relativo apenas a uma notícia.

A variável id representa o identificador único da notícia. Para este recurso estão definidos os métodos GET, PUT e DELETE.

GET
Devolve a notícia com o identificador igual à id.
PUT
Actualiza a notícia com o identificador igual à id, para ficar igual a notícia passada por parâmetro no corpo do pedido HTTP. A notícia fornecida deve seguir o XML Schema noticia.xsd.
DELETE
Remove a notícia com o identificador igual à id. A notícia não é pagada da base de dados, mas sim movida para uma tabela secundária, onde não estará disponível para o público.

/entidades.php

Não se trata de um recurso, mas sim de mais um nível de indirecção adicionado ao URI base. Aponta tanto para os clubes quanto para os seus integrantes.

/entidades.php/clube

Este recurso representa os clubes.

Para este recurso estão disponíveis os métodos GET e POST.

GET
Retorna uma lista de clubes.
POST
Insere um novo clube, enviado em XML no corpo do pedido HTTP. O XML deve respeitar o XML Schema clube.xsd.

/entidades.php/clube/{id}

Este recurso representa um clube.

Este recurso aceita os métodos GET e PUT.

GET
Retorna o clube que tem o identificador igual à id.
PUT
Permite actualizar a informação do clube com o identificador igual à id. A informação deve ser passada em XML no corpo do request HTTP, respeitando o XML Schema clube.xsd.

/entidades.php/clube/{id}/noticias

Este recurso representa as notícias relacionadas com clube com identificador igual à id. Está disponível o método GET.

GET
Retorna as notícias relacionadas com clube com identificador igual à id.

/tempo.php

Este recurso representa as datas que estão associadas à notícias.

Para este recurso e os seus derivados, apenas está definido o método GET. Note que para os seus derivados, os URIs devem seguir o formato especificado abaixo, mas podem ser utilizados os wildcards "ano", "mes" e "dia".

GET
Retorna a lista de datas associadas à notícias.

/tempo.php/{ano}

Este recurso representa as datas associadas às notícias do ano ano.

GET
Retorna as datas associadas às notícias do ano ano.

/tempo.php/{ano}/{mes}

Este recurso representa as datas associadas às notícias do ano ano e mês mes.

GET
Retorna as datas associadas às notícias do ano ano e mês mes.

/tempo.php/{ano}/{mes}/{dia}

Este recurso representa a data associada às notícias do ano ano, mês mes e dia dia.

GET
Retorna a data associada às notícias do ano ano, mês mes e dia dia.
